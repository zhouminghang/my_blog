2.get和post的区别？

回答问题一定要思路清晰，尽量列举，通过对比更能区别两者之间的区别，可以从大小、方式、安全性、使用场景、实现原理等几方面进行发散性思维,最后在讲各自的优缺点就OK了。

如果还想扩展一下，可以扯扯Ajax中get与post的不同，比如Ajax中的get请求会存在缓存问题，也不安全，怎么解决，就给它传递一个获取系统当前时间的函数，让它自动刷新，从而实现清除缓存的问题，而post请求就不会存在这个问题，而且在发送请求的时候get不需要传递一个参数，post需要传递参数；

换可以扩展一下http中的get与post的区别，顺便可以讲讲谷歌浏览器的扩展插件postman，比如之前做的电影项目，就是使用postman进行豆瓣API接口的数据请求

1、get是从服务器上获取数据，post是向服务器传送数据。
2、 get是把参数数据队列加到提交表单的ACTION属性所指的URL中，值和表单内各个字段一一对应，在URL中可以看到。post是通过HTTP post机制，将表单内各个字段与其内容放置在HTML HEADER内一起传送到ACTION属性所指的URL地址。用户看不到这个过程。
3、对于get方式，服务器端用Request.QueryString获取变量的值，对于post方式，服务器端用Request.Form获取提交的数据。
4、 get传送的数据量较小，不能大于2KB。post传送的数据量较大，一般被默认为不受限制。但理论上，IIS4中最大量为80KB，IIS5中为100KB。
5、 get安全性非常低，post安全性较高。但是执行效率却比Post方法好。

建议：

1、get方式的安全性较Post方式要差些，包含机密信息的话，建议用Post数据提交方式；
2、在做数据查询时，建议用Get方式；而在做数据添加、修改或删除时，建议用Post方式；

5.怎么理解闭包？

这个我就凭自己的印象说了，关于闭包的官方定义我记得不太清楚，反正官方定义的概念太过绕弯子，也不好理解

首先我就说说我总结的闭包的三大特证：

1、函数嵌套函数
2、内部函数可以访问外部变量，但外部不能所访问内部变量
3、某个变量长期的贮存在内存中

接下来就巴拉巴拉闭包的好处，可以有效防止变量污染等等；

再说闭包的使用场景，适用于for循环中，具体讲个小demo，比如列表的点击事件弹出相应的索引值;

当然有利必有弊，因为闭包会让某个变量长期贮存再内存中，势必会造成内存的浪费，对于性能肯定有影响，所以要慎用闭包，这样你可以顺便扯扯JavaScript中的垃圾回收机制，甚至可以扯到前端优化这个大的方向，关于前端优化可以自行度娘，大致也就这几方面，比如图片优化，渲染优化，代码优化，css优化，脚本优化，加载优化等等

6.变量的作用域？

我就说了在ES5之前，变量分为局部变量、全局变量。

局部变量就是定义在函数内部的变量，外部无权访问内部的变量；
全局变量就是定义在最外层的变量，或者是定义在函数内部的变量但没有使用var来定义

顺便就说说关于变量的提升，只声明不赋值…

关于作用域还说了ES5中没有块级作用域，在ES6中可以使用let定义一个块级作用域，比如之前的for循环把var改成let；使用const定义一个常量等等

7.ES5与ES6的区别？

这个方向比较大啊，当时没有一点头绪，不知道该从哪方面开始说起，于是乎：我就开始扯了！

先扯了ES5中JavaScript中的五种基本数据类型和一种引用类型，顺便扯了扯两种数据类型一个存放在栈内存一个存放在堆内存中，举了一个小demo讲了讲堆与栈的区别，有点后悔之前学c的时候没好好看指针，讲了讲ES6新增的Symbol数据类型，扯扯位运算；

接着扯ES5之前没有块级作用域，JavaScript中没有函数重载，就是在同一作用域下，同时定义两个相同名称的函数，在调用的时候传递不同个数的参数就会报错，浏览器不知道到底要解析并执行哪个函数，但可以调用函数的arguments对象，来模拟一个伪数组，通过arguments.length获取参数的个数，通过下标可以获取具体的参数，来模拟函数的重载；ES6可使用let、const定义块级作用域

接着在扯ES6中的箭头函数，以及后期开发基于ES6写法的新框架，其他的我也记不太清楚了，就坦白，尴尬的笑笑，面试官也挺和蔼的，碰到我不会的也没说什么，就说想不起来也没关系

8.讲讲AngularJs的第三方插件路由？

首先我说了AngularJs的自带路由ng-route，讲讲具体的配置，然后开始说第三方插件路由ui-router，讲ui-router的强大之处就是可以实现路由的深层嵌套，而自带路由不行，还有就是使用npm或者bower下载安装，导入模块的时候两者的区别等等

这是我之前总结的两者之间的区别：

区别 ng-router ui-router
名称 ng自带路由 第三方路由
功能 能力有限，不能嵌套路由 非常强大，可以循环嵌套路由，实现路由的深层嵌套
下载安装 bower install –save-dev angular-router @x.y.z(版本号，不写默认最新版本) bower install –save-dev angular-ui-route
引入的库 angular-route.min.js angular-ui-router.min.js
占坑标签 <div ng-view></div> <div ui-view></div>
单页面标签 需要加ng-controller=”” 不需要加ng-controller=””
注入依赖模块 [“ngRoute”] [“ui.router”]
相同点 都依赖angularjs库，引到ng后，做单页面应用的时候存在跨域，需要开启服务器

9.讲讲MVVM

虽然之前看过阮一峰老师对MVVM的讲解，但我还是不太理解这个被神化了的MVVM

我就说了首先它是一种开发思想，和MVC开发思想大致一样，M是Model层，主要是模板层;V是View层，即视图层，用户可以直观看到的页面;C是Controller层，控制器层主要用来调度Model层和View层，VM是View-Model层，其实和Controller是一样的；接下来扯扯怎么进行数据的双向绑定，$scope上挂载一些变量，加入到事件队列中，通过脏检查不停的循环检测，当View层数据发生变化就即使更新到Model层的数据库中，当Model层发生变化就即时响应到视图中去
