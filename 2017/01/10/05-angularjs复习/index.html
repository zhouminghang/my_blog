angularjs查漏补缺2

复习

1. AngularJS 是什么？

  • js的一个框架
  • MVC开发思想
  • 模块化
  • 依赖注入
  • 数据的双向绑定
  • 指令
  • 解决了之前对DOM的频繁操作

2. 适用场景：

  • SPa页面（Single Page applactions）
  • CRUD(增删改查)操作的页面

3. MVC是什么？

  • 它是是一种开发思想，而非设计模式
  • model 模型
  • view 视图
  • controller 控制器（调度视图和模型）

4. 模块化

5.依赖注入

6. 数据的双向绑定

执行原理：

（1）.当加载后的angularjs遇到 ng-model = "变量名"，会把它扔到$scope数据库中；

（2）$scope中有一个主进程，监听循环，会一直不停地检测$scope上的变量的值的变化

（3）当检测到相应的$scope.变量名的值发生变化的时候即时响应给页面

（4）而页面中的相应的 ng-model = "变量名" 的值发生变化，也会即时的扔到数据库中

从而实现数据的双向绑定

7. angularJs的简单执行过程

先简单写一下自己的理解，后续上网资料再更一篇详细过程

（1）先解析DOM结构

（2）当加载到angularJs时，会暴露angular对象

（3）html页面完全加载完毕，建立完整的DOM树结构

（4）angularJs正式启动

（5）先去找ng-app, 表明这个模块归angularJs管理

（6）接下来就是angularJs执行该模块的代码

8. angularJs 中的表达式

能够进行简单的操作，不能进行if else的操作

9. 当页面未加载到angularJs的时候，html不会处理，而显示到页面中，因为加载速度比较快，一般不会显示太久，怎么解决未加载angularJs时让它不显示在页面中？

方案一（不推荐）：

  • 将angularJs库引到 <head>标签中，或者紧跟<body>开始标签

方案二：

  • 使用`ng-bind = “变量”，直接加到开始标签内
  • 不用之前的写法

方案三（然并卵）：

  • 使用ng-cloak,
    • 原理：
    • 很简单，就是未加载angularjs前，让它隐藏，加载到angularjs后，angularjs会自行处理，它才会显示
