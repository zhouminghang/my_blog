gulp–构建自动化工程

使用gulp将自己开发的代码压缩到dist目录下（包括html, css, js, lib, fonts, img）

gulp简介：

  • gulp是前端开发过程中对代码进行构建的工具，是自动化项目的构建利器；
  • 它不仅能对网站资源进行优化，而且在开发过程中很多重复的任务能够使用正确的工具自动完成；
  • 使用它，我们不仅可以很愉快的编写代码，而且大大提高我们的工作效率。
  • gulp就是用来把上述流程自动化的，并且现在像gulp和grunt这种工具一般都提供额外的实时刷新服务器，也就是说，在开发过程中监视你的文件改变，实时来帮你刷新浏览器。
  • gulp是基于Nodejs的自动任务运行器，它能自动化地完成 javascript/coffee/sass/less/html/image/css 等文件的的测试、检查、合并、压缩、格式化、浏览器自动刷新、部署文件生成，并监听文件在改动后重复指定的这些步骤。
  • 在实现上，它借鉴了Unix操作系统的管道（pipe）思想，前一级的输出，直接变成后一级的输入，使得在操作上非常简单。通过本文，我们将学习如何使用Gulp来改变开发流程，从而使开发更加快速高效。

下面简单的带大家感受一下gulp的使用：

###1.将自己的整个工程拖到自己的IDE中
（推荐使用sublime，webstorm当然其他编译工具尽量都试试）

###2.以sublime为例，右键打开终端
(1)初始化npm
输入 npm init 一路回车 自动生成package.json文件
（name工程名称， version版本号（注意写成..*），main入口文件， author作者名，license许可证书………. description工程描述 这些都是可写可不写，不写则一路回车即可）
(2)初始化bower
输入 bower init 一路回车 生成bower.json文件

3.手动添加一个 gulpfile.js文件

(1)安装gulp 
     输入  npm install --save gulp  回车
 (2)引用gulp
      在gulpfile.js中引用gulp：
               var gulp = require("gulp");

4.1–压缩html

    (2)在gulpfile.js中引用：

(3)终端执行 gulp html 回车
会自动生成目录dist/index.html文件

4.2–css 如果是sass文件 先转换成css文件，在合并，然后压缩，最后扔进dist/css
(1)终端输入 npm install –save-dev gulp-sass 回车 （将sass转换成css）
npm install –save-dev gulp-concat-css 回车 （将css文件合并）
npm install –save-dev gulp-cssnano 回车 （将css文件进行压缩）
(2)在gulpfile.js中引用：
var sass = require(“gulp-sass”);
var concatCss = require(“gulp-concat-css”);
var cssnano = require(“gulp-cssnano”);

gulp.task("style", function() {
    // 找到所有的sass文件
    gulp.src("css/*.scss")
        // sass转换成css
        .pipe(sass())
        // 合并css，合并为一个main.css
        .pipe(concatCss("main.css"))
        // 压缩css
        .pipe(cssnano())
        .pipe(gulp.dest("dist/css"))
        .pipe(reload({stream: true})); ;
});

(3)终端执行 gulp style 回车
会自动生成目录dist/css/main.css文件

4.3–js文件 – 合并，压缩，混淆

(1)终端输入 npm install –save-dev gulp-concat 回车
npm install –save-dev gulp-uglify 回车
npm install –savedev jshint gulp-jshint 回车

(2)在gulpfile.js中引用：
// js合并，压缩，混淆
var concat = require(“gulp-concat”);
var uglify = require(“gulp-uglify”);
// npm install –save-dev jshint gulp-jshint 两个必须一起安装
var jshint = require(“gulp-jshint”);

gulp.task("sheet", function() {
    gulp.src("js/*js")
    // 合并
    .pipe(concat("app.js"))
    // 压缩
    .pipe(uglify())
    // 混淆
    .pipe(jshint())
    .pipe(gulp.dest("dist/js"))
    .pipe(reload({stream: true})); ;
});

(3)终端执行 gulp sheet 回车
会自动生成目录dist/js/app.js（注意：要将原html中的main.js修改成app.js）

4.4以下只是简单的复制移动图片，文字，库， 类似于4.1
// 4.4.1图片处理 移动
gulp.task(“img”, function() {
gulp.src(“img/.“)
.pipe(gulp.dest(“dist/img”))
.pipe(reload({stream: true}));
});

// 4.4.2.fonts处理 移动
gulp.task("fonts", function() {
    gulp.src("fonts/*.*")
        .pipe(gulp.dest("dist/fonts"))
        .pipe(reload({stream: true}));
});

// 4.4.3.libs处理 
gulp.task("lib", function() {
     多级目录用**（一个即可，即使下面有多级目录）
    gulp.src("lib/**/*.*")
        .pipe(gulp.dest("dist/lib"))
        .pipe(reload({stream: true}));
});

4.5 编译任务

gulp.task("build", ["style", "sheet", "img", "html", "lib", "fonts"]);

4.6 开启服务

  • (1)终端输入 npm install –save-dev browser-sync 回车

  • (2)在gulpfile.js中引用：
    var browserSync = require(‘browser-sync’).create();
    var reload = browserSync.reload;

    gulp.task(“serve”, [“build”], function() {

    browserSync.init({
    server: {
        baseDir: "dist/"
    }
});

    //监听文件变化

    gulp.watch("css/*.css", ["style"]);
gulp.watch("js/*.js", ["sheet"]);
gulp.watch("img/*.*", ["img"]);
gulp.watch("fonts/*.*", ["fonts"]);
gulp.watch("lib/**/*.*", ["lib"]);

    });
    (3)终端开启服务 gulp serve 回车

         会自动在locaoihost:3000（3000是默认端口，如果页面显示请求失败要么是你的html文件名称不是index.html，要么是路径含有中文）自动打开你的工程

(4)退出服务  Ctrl+c   y  回车
